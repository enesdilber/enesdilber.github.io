How to decide Model Complexity?¶

A key problem in statistical modeling is model selection, how to choose a model at an appropriate level of complexity.

  • How many classes should I use in my mixture model?
  • How many factors should I use in factor analysis?
  • Model selection metrics: goodness of fit and complexity penalty

Bayesian Non-parametric Methods¶

  • Rather than comparing models that vary in complexity, the BNP approach is to fit a single model that can adapt its complexity to the data
  • Furthermore, BNP models allow the complexity to grow as more data are observed, such as when using a model to perform prediction
  • Each model expresses a generative process of the data that includes hidden variables
  • Hidden structure in model is assumed to grow with the data. That distinguishes Bayesian nonparametric models from other Bayesian.
  • Number of clusters from the data and allows the number of clusters to grow as new data points are observed

  • $\textbf{Bayesian}:$

    • $ P(parameters|data) \propto P(data|parameters)P(parameters) $

  • $\textbf{Non-parametric}:$

    • Not-finite parameters, unbounded/growing/infinite number of parameters

Trying to find the latent structure of data.

Lets do not consider $\sigma^2_k$ for simplicity. Here is our Gaussian Mixture Model:

  • k = 2 (2 clusters)
  • Prior of each cluster means: $\mu_k \overset{iid}\sim N(\mu_0, \Sigma) $
  • Prior percentage of points from first cluster $p_1 \sim Beta(a_1, a_2)$, $ \ p_2 = 1 - p_1$
  • Probability of each data point belongs to $cluster \ 1$ or $2$ is $z_n \overset{iid}\sim Categorical(p_1, p_2)$
  • Distribution of each data point $x_n \overset{iid}\sim N(\mu_{z_n}, \Sigma) $

Choosin $K = \infty $ in Drichlet

$$p_{1:k} \sim Drichlet(a_{1:k})$$ is equivalent to: $$V_1 \sim Beta(a_1, \sum_{k=2}^{K}a_k) \ \ \ \ \ p_1 = V_1$$ $$V_2 \sim Beta(a_2, \sum_{k=3}^{K}a_k) \ \ \ \ \ p_2 = (1-p_1) V_2$$ $$.$$ $$.$$ $$.$$ $$V_{k-1} \sim Beta(a_{k-1}, a_k) \ \ \ \ \ p_{k-1} = (1-\sum_{i = 1}^{k-2}p_i)V_{k-1}$$ $$p_k = 1-\sum_{i = 1}^{k-1}p_i$$

In general: $$\large{G = \sum^{\infty}_{i=1}\pi_i \delta_{\phi_k}}$$

  • $\pi \sim GEM(\alpha)$
  • $\phi_k \overset{iid}\sim G_0$
  • $G \sim DP(\alpha, G_0)$
  • $\alpha$ is parameter of stick-breaking, controls sparsity
  • $G_0$ base measure that gives location of the atoms Draw from G is a draw with location at $\phi$ space and direction $\pi$. The important thing is direction is independent from location.

Measure in Dirichlet Process¶

$$\large{G(A) = \sum^{\infty}_{i=1}\pi_i \delta_{\phi_k}(A) = \sum_{k:\phi_k \in A} \pi_k}$$

  • A discrete Measure
  • Stochastic Process with index set = {$G(A): A \in \Omega$}

Example: Let's draw from $G \sim DP(\alpha, N_2(\mu_0, \Sigma_0))$¶

Let:
$p = (p_1, p_2, ...)\sim GEM(\alpha)$ Draw from GEM

$\mu_k \overset{iid} \sim N(\mu_0, \Sigma_0), k=1,2,...$ Draw from Cluster Location

Sampling from $G$ is equivelent to these steps:

$z_n \overset{iid} \sim Categorical(p) $ This data point blogs to selected cluster

$\mu_n^* = \mu_{z_n} $, $n^{th}$ data point will be around the mean indexed by $z_n$

$i.e. \mu_n^* \overset{iid}\sim G$

Since it is a discrete distribution, the clusters will be repeated a lot and will get data points around some clusters. At the end we sample data point, from:

$x_n \overset{indep}\sim N(\mu_n^*, \Sigma) $
